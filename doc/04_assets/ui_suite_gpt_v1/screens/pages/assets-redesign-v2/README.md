# 资产台账 UI 重设计 v2

> 状态：视觉探索稿，已于 2026-07-13 的契约审查中判定为 **不得直接替换现有目标图、不得直接进入前端实现**。原因是本稿把现行五类资产 Tab 和五个详情状态改造成了新的任务态/上下文信息架构。正式契约对齐稿见相邻目录 `assets-redesign-v3-contract-aligned/`。

更新时间：2026-07-13

本目录基于“资产研判工作台”方向重做资产台账页面。新版不再把“终端、服务器、网络设备、业务系统、未知资产”全部作为同一级 Tab，而是采用统一资产对象模型。

## 信息架构

- 顶层任务态：全部资产、待确认、高风险、最近变更。
- 资产类型：终端、服务器、网络设备等，作为筛选条件。
- 未知资产：作为“待确认”生命周期状态进入归属确认队列。
- 业务系统：作为资产关系实体，在关系上下文中呈现。
- 资产上下文：概览、暴露面、关系、行为、证据、变更。
- 选中资产：切换任务态或上下文时持续保留。

## 实现约束

1. 所有页面保持相同的 AppShell、筛选区、表格密度和资产上下文结构。
2. 前端必须使用真实 React DOM/CSS 实现，不得把整页 PNG 作为页面覆盖层或背景。
3. 右侧上下文仅随所选资产和上下文 Tab 变化，不重复加载整页。
4. 主操作每个状态只保留一个，其他动作降级为辅助按钮或更多菜单。
5. 状态不能仅依赖颜色表达，需要同时使用图标、文字或形状。
6. 图中日期和数据用于设计示例，真实实现应由 API 返回。
