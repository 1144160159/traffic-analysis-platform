## 一、重构原则

- 将原先“工程化增强、第二轮、第三轮、第四轮、第五轮、第六轮”等过程型章节，归并为规范性文档章节。
- 产品设计文档采用 PRD/产品规格说明书结构，重点回答“给谁用、解决什么问题、有哪些页面和功能、如何验收”。

- 两份文档通过需求追溯、字段口径、接口契约、测试验收四类桥接点保持一致。

## 四、旧内容归并规则

| 旧章节类型 | 归并到产品设计 | 归并到技术设计 |
|---|---|---|
| 工程化增强设计 | 产品能力、信息架构、验收章节 | 总体架构、数据流、部署运行章节 |
| 第二轮完整性扩充 | 功能需求、页面交互、验收脚本 | 子系统详细设计、数据血缘、非功能设计 |
| 第三轮业务细节 | 用户场景、告警 SOP、数据口径 | 业务流程到技术链路映射、业务规则 |
| 第四轮产品深化/接口扩展 | 产品定位、角色、页面状态、体验规范 | 接口契约、REST/gRPC/Kafka、端到端流程 |
| 第五轮产品优化 | 页面规格、状态机、字段口径、权限审计 | 前后端接口对齐、内部实现分层 |
| 第六轮产品优化 | 读者路径、原型级页面、运营闭环、评审门禁 | 测试验收、运维交付、风险演进 |
