# 数据采集分工设计

基于《任务书》和《实施方案》中课题一"园区网络流量智能检测与分析"的多源异构数据融合要求。

## 一、文档依据

> 实施方案原文：
> "研究基于多源异构数据融合的复杂恶意攻击行为检测技术，将园区网络中的
>  **资产信息、设备日志、用户行为、流量数据**等多源异构数据融合，
>  支持对网络潜在的复杂恶意攻击行为进行智能检测。"

## 二、数据采集子系统归属

```

```

## 三、归属明细

### 3.1 流量数据 → `rust/probe-agent` ✅ 已完成

| 路径 | 说明 |
|------|------|
| `rust/probe-agent/probe-agent/src/` | 探针核心代码，42个模块 37k行 |
| 采集方式 | AF_XDP/AF_PACKET 网卡实时抓包 + PCAP 离线回放 |
| 输出 | FlowEvent (Protobuf) → gRPC → Ingest Gateway → Kafka |
| 配置 | `config.yaml`，支持在线/离线两种模式 |

### 3.2 资产信息 → `go/control-plane/cmd/asset-service`（新建）

| 路径 | 说明 |
|------|------|
| `go/control-plane/cmd/asset-service/main.go` | 服务入口 |
| `go/control-plane/internal/asset/api/` | gRPC/HTTP API |
| `go/control-plane/internal/asset/service/` | 资产发现/管理逻辑 |
| `go/control-plane/internal/asset/repository/` | PostgreSQL 数据访问 |
| `proto/traffic/v1/asset.proto` | 资产 Protobuf 定义 |

**资产数据来源：**
- 被动：探针上报的 MAC 地址 → OUI 厂商识别
- 被动：Ingest Gateway 接收的流量中提取 IP↔MAC 映射
- 主动（可选）：SNMP/LLDP 网络发现
- 人工：Web UI 资产台账录入

**核心数据表：**
```sql
-- assets 资产表
CREATE TABLE assets (
    asset_id    UUID PRIMARY KEY,
    tenant_id   VARCHAR(64) NOT NULL,
    ip_address  VARCHAR(45),
    mac_address VARCHAR(17),
    hostname    VARCHAR(255),
    vendor      VARCHAR(128),        -- OUI 厂商
    os_type     VARCHAR(64),         -- 操作系统猜测
    first_seen  TIMESTAMP,
    last_seen   TIMESTAMP,
    tags        JSONB,
    UNIQUE(tenant_id, mac_address)
);

-- asset_events 资产变更事件表
CREATE TABLE asset_events (
    id          BIGSERIAL PRIMARY KEY,
    asset_id    UUID REFERENCES assets(id),
    event_type  VARCHAR(32),         -- 'first_seen', 'ip_changed', 'inactive'
    old_value   JSONB,
    new_value   JSONB,
    created_at  TIMESTAMP DEFAULT NOW()
);
```

### 3.3 设备日志 → 采集层 `Fluent Bit` + 处理层 `java/flink-jobs/flink-log-job`（新建）

| 路径 | 说明 |
|------|------|
| `deployments/k8s/fluent-bit/` | Fluent Bit DaemonSet 部署配置 |
| `java/flink-jobs/flink-log-job/` | Flink 日志富化作业 |
| 存储 | Loki（已部署，日志存储）+ OpenSearch（已部署，全文检索） |

**日志采集管道：**
```
设备 Syslog (UDP 514)
  → Fluent Bit (DaemonSet, 每节点)
  → Kafka topic: device.logs.v1
  → Flink Log Job (解析/富化/添加 tenant_id)
  → Loki (存储)
  → OpenSearch (索引，供告警查询)
```

**Flink Log Job 职责：**
- 解析 Syslog RFC 5424/3164 格式
- 提取设备 IP、时间戳、严重级别、消息体
- 关联 asset_service 获取设备归属
- 写入 Loki（日志存储）+ OpenSearch（全文索引）

### 3.4 用户行为 → `go/control-plane/cmd/auth-service`（扩展） + `iam/`（已部署）

| 路径 | 说明 |
|------|------|
| `go/control-plane/cmd/auth-service/` | 认证服务（扩展 Event Listener） |
| `iam/` (Keycloak) | 已部署，需配置 Kafka Event Publisher |
| 存储 | OpenSearch（审计索引）+ Loki（登录日志） |

**用户行为数据流：**
```
Keycloak (用户登录/权限变更)
  → Kafka topic: user.events.v1
  → Flink User Behavior Job
  → OpenSearch 审计索引

APISIX (API 访问审计)
  → Loki (已配置)
  → 需新增: Kafka topic + Flink 会话关联
```
