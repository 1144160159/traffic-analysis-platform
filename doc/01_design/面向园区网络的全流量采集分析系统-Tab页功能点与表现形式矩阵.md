# 面向园区网络的全流量采集分析系统 Tab 页功能点与表现形式矩阵

更新日期：2026-06-27

本文档从页面内 Tab 状态出发，补充 `面向园区网络的全流量采集分析系统-二级菜单功能点与表现形式矩阵.md`。它用于后续 UI 图补生成和验收，不改变左侧导航结构。

适用范围：

- 现役 UI 图范围为 `doc/04_assets/ui_suite_gpt_v1` 的 163 张；`/topics` 专题面板是现役前端菜单和路由，但不生成单张 `topics.png`。
- 加密隧道、数据外传、APT 战役是 `/topics` 专题面板内的业务切换状态，三张拆分设计输入必须在前端合并为同一个 `/topics` 页面，不拆成三个左侧菜单，也不作为三个独立页面主图。
- 后续任何页面如果 UI 设计图按 Tab 拆多张，也必须合并为一个路由页面内的 Tab/Segmented 状态；除非产品文档明确要求新增导航入口，否则不得新增左侧菜单或独立业务路由。
- 本文中的 `Topic 健康` 指 Kafka Topic 数据质量状态，不等同于 `/topics` 专题面板。
- 告警详情、战役详情等详情路由不新增左侧二级菜单，左侧仍继承来源二级菜单高亮。
- 资产台账层级修正（2026-06-27）：`资产类型 Tab` 是 `/assets` 页面级大 Tab，用于切换终端、服务器、网络设备、业务系统、未知资产五类资产工作台；`资产详情小 Tab` 是选中单个资产后的局部详情组件，不与资产类型 Tab 平级，不生成完整页面公共区域。

## 1. 对现有二级菜单矩阵的差异结论

| 类型 | 涉及页面 | 结论 | 处理方式 |
|---|---|---|---|
| 源矩阵已覆盖功能点，但未拆 Tab 状态图 | 数据质量、加密流量、资产台账、实体图谱、行为基准、规则管理、模型管理、白名单、审计日志、战役详情 | 大部分能力在原矩阵中已有功能点和表现形式，但没有按页面内 Tab 生成独立状态图 | 本文按 Tab 维度补充生图矩阵，后续纳入 UI 图队列 |
| 源矩阵功能点漏项或表达不足 | 数据质量、资产台账、行为基准、规则管理、模型管理 | `质量报告`、`质量设置`、`未知资产`、`协议基线`、`时间段基线`、`依赖引用`、`样本类型拆分`、`审计门禁`未被显式拆出 | 需要在后续修订二级菜单矩阵时同步补充 |
| 详情路由未单列 | 告警详情、战役详情 | 原矩阵以告警中心、战役列表和取证分析覆盖相关能力，但没有把详情页 Tab 作为独立 UI 状态管理 | 详情页不新增左侧菜单，只补 Tab 状态图和表现形式 |
| 专题面板页内模式 | 专题面板 `/topics` | 原二级菜单矩阵中的加密隧道、数据外传、APT 战役能力统一收敛到单个专题面板页面 | 三张 Tab 设计输入合并为 `/topics` 页内模式，左侧菜单只保留“专题面板” |

## 3. 每个 Tab 页的内容与表现形式明细

读法：

- `页面内容（必须画出来）` 是该 Tab 激活后主工作区中必须出现的数据对象和业务信息，不允许只改变 Tab 标题。
- `表现形式（怎么画）` 是该 Tab 的主要组件组合，用于约束 imagegen 生成时的画面结构。
- 同一页面的其他公共区、左侧菜单、顶部、底部和非当前 Tab 的上下文保持不变，只替换当前 Tab 对应的业务内容。
- 资产台账的 `终端 / 服务器 / 网络设备 / 业务系统 / 未知资产` 属于页面级大 Tab，每张图都应保留 `/assets` 页面公共工作台结构，并在右侧只放当前选中对象摘要，不放资产详情内部小 Tab。
- 资产详情的 `基础信息 / 网络接口 / 开放服务 / 归属信息 / 历史变更` 属于局部小 Tab，只画资产详情组件本体，不画 AppShell、资产台账列表、页面级资产类型 Tab、筛选区或统计区。

## 4. Tab 页生成验收口径

每张 Tab 变体图必须同时满足以下条件：

1. 当前 Tab 激活态清晰，未激活 Tab 仍可见，Tab 名称必须与矩阵一致。
2. 主工作区必须能直接看出该 Tab 的业务主题，例如 `Topic 健康` 必须出现 Kafka offset、积压、消费延迟或分区倾斜，不得只画通用质量卡片。
3. `页面内容（必须画出来）` 中的关键对象至少覆盖 3 类；对象数量不足时，用趋势、明细、右侧详情或异常原因补足信息密度。
4. `表现形式（怎么画）` 中的组件类型必须作为主视觉出现；如果写了热力图、关系图、时间线、Diff、状态机，不能只用普通表格替代。
5. 每个 Tab 都要保留闭环动作入口，例如跳转、导出、修复任务、审批、生成工单、查看证据或进入详情抽屉。
6. 详情路由 Tab 只改变详情页内部内容，不新增左侧二级菜单，不改变来源菜单继承高亮。

## 5. 后续补图生成规则

1. Tab 变体不新增左侧二级菜单；只改变主工作区内的当前 Tab 和业务内容。
2. 顶部单栏、左侧单栏、底部单栏必须继续以 `screen.png` 为唯一公共区基准。
3. 详情路由继承来源菜单高亮：告警详情继承 `告警中心`，战役详情继承 `战役列表`。
4. 每张 Tab 变体必须保持同一页面的信息架构，只替换 Tab 内业务内容，不破坏周边 KPI、筛选区和右侧闭环区的上下文。
5. 后续进入 imagegen 队列时，仍按每 3 张生成后更新一次上下文摘要。
6. 保存 imagegen 结果时继续使用 `doc/04_assets/ui_suite_gpt_v1/extract_latest_imagegen.py <targetFile>`，目标文件建议落到 `doc/04_assets/ui_suite_gpt_v1/screens/pages/`。
